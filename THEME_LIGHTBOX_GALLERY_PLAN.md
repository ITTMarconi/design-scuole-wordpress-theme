# Plan — custom theme support for imported article galleries

## Goal
Add frontend behavior in the custom WordPress theme so the galleries generated by `upload-articles` open a lightbox with previous/next navigation (carousel behavior).

## Imported markup to support
The importer now produces this structure for articles that have **no explicit image placeholders** and therefore append images at the end:

```html
<section class="ittm-lightbox-gallery" data-ittm-gallery="post-slug" aria-label="Galleria immagini: Article title">
  <figure class="ittm-lightbox-gallery__item">
    <a href="https://site/wp-content/uploads/.../image-1.webp"
       class="ittm-lightbox-gallery__link"
       data-ittm-lightbox-trigger
       data-ittm-gallery-index="1">
      <img src="https://site/wp-content/uploads/.../image-1.webp"
           alt="Article title"
           class="ittm-lightbox-gallery__image wp-image-123">
    </a>
  </figure>

  <figure class="ittm-lightbox-gallery__item">...</figure>
</section>
```

Notes:
- `data-ittm-gallery` groups all images belonging to the same article/gallery.
- Each link already points to the full media file URL.
- Each image has the normal WP media class `wp-image-<id>`.
- Cover images are excluded from this gallery.
- If an article uses image placeholders in the body, the current importer does **not** generate this gallery wrapper.

## Recommended implementation
Use a lightweight lightbox/carousel library in the theme, or implement a small custom modal.

### Preferred behavior
- Clicking any `.ittm-lightbox-gallery__link` opens a modal/lightbox.
- The modal shows the clicked image first.
- Users can move to the next/previous image in the same gallery.
- Keyboard support:
  - `Esc` closes
  - `←` previous
  - `→` next
- Optional but recommended:
  - click backdrop to close
  - swipe on mobile
  - focus trap for accessibility

## Suggested task breakdown

### 1. Add theme assets
- Add a JS file for the gallery/lightbox behavior.
- Add a CSS file for gallery layout and modal styling.
- Enqueue both only on singular content pages where article content is rendered.

### 2. Enhance the gallery grid
Style these selectors:
- `.ittm-lightbox-gallery`
- `.ittm-lightbox-gallery__item`
- `.ittm-lightbox-gallery__link`
- `.ittm-lightbox-gallery__image`

Suggested layout:
- CSS grid
- 2 columns on mobile-small/medium
- 3+ columns on larger screens
- consistent gaps
- image thumbnails cropped with `aspect-ratio` or object-fit

### 3. Build the lightbox behavior
JS should:
1. Find every `.ittm-lightbox-gallery`
2. Collect its child links `.ittm-lightbox-gallery__link`
3. On click, prevent default navigation
4. Open modal with the clicked image index
5. Support next/previous within the same gallery only

### 4. Accessibility requirements
- Modal should have `role="dialog"` and `aria-modal="true"`
- Add accessible labels for close/prev/next buttons
- Restore focus to the clicked thumbnail when the modal closes
- Prevent background scroll while open

### 5. Progressive enhancement
If JS fails or is disabled, links should still open the raw media file normally.

## Acceptance criteria
- Imported end-of-article galleries render as a visible thumbnail grid.
- Clicking one thumbnail opens a lightbox instead of navigating away.
- The lightbox supports previous/next image navigation for that gallery.
- Multiple galleries on the same page do not mix images across groups.
- Without JS, clicking a thumbnail still opens the media URL.

## Nice-to-have extras
- Caption support later if importer starts emitting captions
- image counter (e.g. `2 / 8`)
- touch swipe support
- preload adjacent images for smoother navigation

## Integration hint for the theme agent
The safest integration point is the rendered post content. Do not rewrite the imported markup server-side unless necessary; prefer unobtrusive JS that enhances the existing HTML structure.
